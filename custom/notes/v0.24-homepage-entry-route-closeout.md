# 版本记录：v0.24-homepage-entry-route-closeout

## 1. 本轮目标

本轮目标是完成首页入口与栏目路由收口，确保首页已经出现的所有核心入口都能正常点击进入，不出现 404、不出现空链接、不跳错栏目。

这一轮优先级属于：

- 功能补充
- 交付收口
- 可维护性提升

这件事之所以必须优先处理，是因为首页已经调整成完整行业入口结构，但如果入口点进去仍然报错，就会直接破坏“可交付”和“可正式商用”的基础体验。

## 2. 背景与上下文

本轮承接以下记录：

- `v0.22-knowledge-dev-timeout`
- `v0.23-homepage-industry-entry-completion`

修改前的状态是：

1. 首页的行业入口结构已经成立，首屏、七大入口、热门问题、词库精选、知识区、品牌区、信任体系、合作区、最新内容区都已经存在。
2. 但首页中有多类入口仍然只是语义表达，还没有落到稳定路由上。
3. 已有主栏目路由可以复用：
   - `/knowledge`
   - `/terms`
   - `/brands`
   - `/admin/login`
4. 榜单、指数、标准、智库、趋势、事件、品牌进展、合作等入口，如果不补落地页，就会形成交付级缺口。

已知约束：

- 不改数据库结构
- 不扩展后台
- 不做复杂数据模型
- 不做大视觉重构
- 只补“能进入”的最小页面与正确路由

## 3. 问题分析

表面现象：

- 首页结构已经完整，但部分入口只能停留在文案层面。
- 部分入口如果直接复用旧链接，会跳到不够准确的栏目，或者只能落到锚点，不满足“栏目可进入”的交付要求。

核心矛盾：

- 一方面，首页必须把行业趋势、行业事件、品牌进展、中眠榜、中眠指数、睡眠标准、中眠智库、合作入口显性摆出来。
- 另一方面，这一轮又不能扩复杂模型，也不能为每个栏目做重后台和真内容链路。

主要原因：

1. 首页结构先于栏目落地页完成，导致入口语义先成立，但路由未同步收口。
2. 现有三条主线页面足以承接知识、词库、品牌三类入口，但不足以自然承接全部行业参考入口。
3. 如果直接把所有入口都硬塞进 `/knowledge` 或 `/brands`，会造成“能点开，但栏目语义错误”的问题。

## 4. 候选方案比较

### 方案 A：所有入口都并入已有栏目

核心思路：

- 趋势、事件、品牌进展、智库、榜单、指数、标准全部继续指向 `/knowledge`、`/brands` 或首页锚点。

优点：

- 改动最小
- 不需要新增路由

缺点：

- 栏目语义不准确
- 用户点击后容易误判栏目边界
- “中眠榜 / 中眠指数 / 睡眠标准 / 中眠智库”仍然不是真正可进入的栏目

放弃原因：

- 不符合本轮“首页出现的所有入口都必须能正常进入”的要求。

### 方案 B：为每个栏目单独新建独立页面文件

核心思路：

- 新建 `/trends`、`/events`、`/brand-progress`、`/rankings`、`/indexes`、`/standards`、`/think-tank`、`/cooperation` 八个独立页面。

优点：

- 每个栏目边界最清楚
- 后续可单页独立扩展

缺点：

- 文件数量明显增加
- 绝大部分页面结构高度重复
- 对当前“最小改动”原则不够友好

放弃原因：

- 在当前阶段收益不如统一收口方案高。

### 方案 C：保留已有主栏目 + 用一个最小动态栏目页承接剩余入口

核心思路：

- 继续复用已有主栏目：
  - `/knowledge`
  - `/terms`
  - `/brands`
  - `/admin/login`
- 用一个动态栏目页 `app/(site)/[channel]/page.tsx` 承接剩余固定栏目：
  - `/trends`
  - `/events`
  - `/brand-progress`
  - `/rankings`
  - `/indexes`
  - `/standards`
  - `/think-tank`
  - `/cooperation`
- 对未知 slug 直接 `notFound()`

优点：

- 新增文件最少
- 所有首页入口都能稳定进入
- 栏目语义清楚
- 页面结构统一、后续好扩展

缺点：

- 多个栏目共用一份配置式页面，不如完全独立页面灵活
- 以后如果栏目内容差异变大，仍可能拆回独立页面

最终选择原因：

- 这是本轮“最小改动、最大收益、交付可用”的最佳平衡方案。

## 5. 最终决策

本轮最终采用方案 C。

关键决策如下：

1. 保留已有主栏目，不推翻原有知识、词库、品牌和后台登录入口。
2. 用一个动态栏目页承接首页剩余行业入口，避免一口气创建 8 个重复页面文件。
3. 每个新栏目页必须具备：
   - 页面标题
   - 一句话定位
   - 这个栏目解决什么问题
   - 当前可进入的相关栏目推荐
   - 专业建设中说明
4. 同步修正首页所有对应链接，确保首页主要入口点进去不再 404。
5. 热门问题入口继续合并到 `/knowledge`，但改成携带搜索参数的明确入口，而不是无差别跳知识首页。

## 6. 具体实现

本轮实际改动如下。

### 1. `app/(site)/page.tsx`

作用：

- 重写首页中的入口配置与跳转关系
- 把原来不够准确的首页入口统一收口到正确栏目

具体改动：

1. 七大核心入口改为：
   - 睡眠知识 -> `/knowledge`
   - 睡眠词库 -> `/terms`
   - 睡眠品牌 -> `/brands`
   - 行业趋势 -> `/trends`
   - 行业事件 -> `/events`
   - 品牌进展 -> `/brand-progress`
   - 中眠智库 / 行业参考 -> `/think-tank`
2. 信任体系区改为真实可点击：
   - 中眠榜 -> `/rankings`
   - 中眠指数 -> `/indexes`
   - 睡眠标准 -> `/standards`
   - 中眠智库 -> `/think-tank`
3. 企业合作区改为真实可点击：
   - 品牌收录 -> `/cooperation`
   - 行业合作 -> `/cooperation`
   - 榜单 / 指数 / 智库合作 -> `/cooperation`
4. 热门问题入口从“统一跳 `/knowledge`”改为“跳 `/knowledge?q=问题词`”。
5. 品牌区的“行业事件”“品牌进展”说明卡补了进入对应栏目的按钮。
6. 首页仍保留真实数据优先 + 结构化兜底的行业入口表达。

### 2. `app/(site)/[channel]/page.tsx`

作用：

- 承接首页新增的固定栏目路由

具体实现：

- 通过配置对象统一定义每个栏目：
  - 标题
  - 定位文案
  - 栏目作用说明
  - 相关栏目推荐
  - metadata 路径和关键词
- 通过 `generateStaticParams` 只开放八个固定栏目
- 通过 `dynamicParams = false` 和 `notFound()` 防止未知栏目误入

承接的栏目包括：

- `/trends`
- `/events`
- `/brand-progress`
- `/rankings`
- `/indexes`
- `/standards`
- `/think-tank`
- `/cooperation`

### 3. `AGENTS.md`

作用：

- 将“当前最新一期正式记录”更新到本轮记录，保证后续任务优先读取本轮结论。

## 7. 本轮优点

本轮方案相比之前带来的正向效果：

1. 首页所有核心入口都已有稳定去向，不再停留在结构表达层。
2. 首页点击后不再容易落入错误栏目。
3. 行业趋势、行业事件、品牌进展、智库、榜单、指数、标准、合作入口都已具备“可进入”的专业栏目页。
4. 没有改数据库、没有改后台、没有引入复杂模型。
5. 用一个动态栏目页统一收口，避免了重复页面文件堆积。
6. 热门问题入口变得更像真实入口，而不是“点什么都回知识首页”。

## 8. 本轮缺点与代价

必须如实说明本轮代价：

1. 新增的栏目页目前仍是“结构化栏目页”，不是完整真内容频道。
2. `/trends`、`/events`、`/brand-progress`、`/rankings`、`/indexes`、`/standards`、`/think-tank`、`/cooperation` 还没有独立真实数据模型。
3. 当前多个栏目共用一份配置式页面，未来如果栏目差异增大，可能需要拆回独立实现。
4. 首页这轮只是收口路由，不涉及更深层的栏目运营配置。

这些都不是当前阻断，但属于后续可能继续演进的方向。

- 这不是本轮首页入口路由收口引入的新问题，而是当前本地数据库运行态问题。

## 10. 本轮结论

本轮最重要的结论是：

1. 首页结构完整之后，下一步必须把入口路由收口，否则“看起来完整”不等于“可交付”。
2. 在不改模型、不加后台的前提下，用“已有主栏目 + 最小动态栏目页”是收口首页入口的最优解。
3. 当前中眠网首页已经满足“首页入口点进去不 404、能进入清楚栏目页或专业占位页”的交付要求。

## 11. 下一轮建议

下一步最值得继续做的 1~3 件事：

1. 如果用户认可当前首页入口与栏目落地方式，可以整理当前工作区，做一次清晰提交。
2. 如果后续要继续做正式上线准备，可逐步把 `/trends`、`/events`、`/brand-progress`、`/think-tank` 等栏目从结构页升级为真实内容频道。
3. 如果本地数据库稳定恢复，可再做一轮首页与三条主线的联动回归，确认首页中段真实数据占比是否需要进一步提升。
