# 版本记录：v0.4-前台行业入口结构重构

## 1. 本轮目标

本轮目标不是继续打磨视觉，也不是推进后台 CRUD，而是先把中眠网前台从“有门户感的内容站”重构成“显性成立的中国睡眠产业入口”。

本轮重点是：

- 重构首页信息架构
- 显性呈现基础知识入口、信任入口、商业入口三层能力
- 重构词库页为词条系统
- 重构知识页为问题入口 + 解决路径
- 强化品牌页的行业名录、信任展示和商业承接结构

本轮属于：

- 信息架构重构
- 产品定位纠偏
- 前台入口结构定型

## 2. 背景与上下文

### 2.1 本轮读取内容

本轮开始前读取了：

1. `AGENTS.md`
2. `custom/notes/v0.3-prisma-schema-foundation.md`
3. `app/(site)/page.tsx`
4. `app/(site)/terms/page.tsx`
5. `app/(site)/knowledge/page.tsx`
6. `app/(site)/brands/page.tsx`

### 2.2 与上一轮关系

本轮不是推翻产品定位，而是纠偏首页表达方式。

延续的旧思路：

- 中眠网不是普通资讯站
- 中眠网必须同时承担知识入口、信任入口、商业入口
- 词库、知识、品牌仍然是一期核心主线

本轮修正的旧判断：

- 之前首页虽然有门户感，但表达方式偏“内容门户”
- 这会让中眠网在第一眼更像资讯站，而不像行业入口
- 因此本轮不再让内容流主导首页，而是让行业入口结构主导首页

## 3. 问题分析

### 3.1 当前核心问题

当前 demo 的问题不是没有板块，也不是没有信息密度，而是：

- 版块组织还偏内容流
- 用户第一眼无法迅速看懂中眠网的三层能力
- 榜单、指数、标准、智库仍被表达成“未来入口”，而不是当前结构的一部分

### 3.2 真正要解决的矛盾

这一轮真正要解决的是：

- 如何让首页从“内容聚合页”变成“行业入口页”
- 如何让词库、知识、品牌不再像三个平行频道，而成为不同层级的入口结构
- 如何在内容量还不多时，先把行业信任体系和商业承接体系摆出来

## 4. 候选方案比较

### 方案 A：保留旧首页，只微调文案和排序

**核心思路**

保留原有结构，稍微把文案改得更像行业门户。

**优点**

- 改动小
- 视觉延续成本低

**缺点**

- 根本问题不会被解决
- 用户仍会优先感知到内容流而不是行业入口结构

**结论**

不选。只是修辞变化，不是结构变化。

### 方案 B：继续让推荐内容和最新内容主导首页

**核心思路**

保留内容流在首页前半段的高权重，只是把品牌和服务入口稍微提前。

**优点**

- 首页看起来比较“丰富”
- 和传统资讯门户接近

**缺点**

- 很容易继续滑向资讯站
- 中眠网的行业入口定位不够显性

**结论**

不选。与本轮目标相冲突。

### 方案 C：首页先讲平台是什么，再讲三层能力，最后才放内容流

**核心思路**

把首页彻底改成：

1. 平台定位 + 七个总入口  
2. 基础知识入口  
3. 信任入口  
4. 商业入口  
5. 内容流降权放后半段

**优点**

- 用户更容易在 3 秒内理解平台定位
- 首页直接具备行业入口结构
- 避免继续像资讯站

**缺点**

- 首页的“内容门户感”会被主动压低
- 需要接受内容流不再是首页主角

**结论**

最终采用本方案。

## 5. 最终决策

### 5.1 首页新排序

首页改为以下顺序：

1. 平台定位 + 核心总入口
2. 基础知识入口
3. 信任入口
4. 商业入口
5. 内容流

### 5.2 各模块目标说明

#### 第一屏：平台定位 + 核心总入口

目标：

- 让用户立即知道中眠网是什么
- 让七个入口在第一屏就成立
- 告诉用户这里不是普通资讯站，而是行业入口系统

#### 第二层：基础知识入口

目标：

- 先把中眠网的知识基础设施身份立住
- 强调词库、问题入口、分类入口、专题入口
- 明确“先查知识，再看内容”

#### 第三层：信任入口

目标：

- 让品牌、榜单、指数、标准、智库提前成为信任体系
- 强化行业参考属性
- 让用户感知平台具备参考价值，而不是单纯阅读价值

#### 第四层：商业入口

目标：

- 让企业和机构知道中眠网未来能承接合作
- 让商业入口在首页就显性成立
- 避免首页只服务普通读者，不服务行业合作方

#### 第五层：内容流

目标：

- 保留内容活跃度与更新感
- 但明确降权，不再主导首页表达

## 6. 具体实现

### 6.1 首页重构

首页已改为行业入口结构：

- 第一屏直接写明：
  “中国睡眠产业的知识入口、品牌入口、信任入口与服务入口”
- 第一屏下方直接放出七个核心总入口，并逐个解释它们代表什么
- 基础知识入口独立成层
- 信任入口独立成层
- 商业入口独立成层
- 推荐内容 / 最新内容放到首页后半段

### 6.2 词库页重构

词库页已从“频道列表页”改成“词条系统页”表达：

- 强调词条系统、定义、节点、分类
- 加入分类目录层
- 词条卡片突出：
  - 词条名
  - 一句话定义
  - 所属分类
  - 关联节点数量
  - 相关入口说明

### 6.3 知识页重构

知识页已从“知识内容列表”改成“问题入口 + 解决路径”表达：

- 先展示问题分类
- 再展示人群 / 场景分类
- 再展示“问题—原因—建议—延伸阅读”结构
- 主动压低新闻感和文章流感

### 6.4 品牌页重构

品牌页已从“品牌卡片页”改成“行业名录 + 信任展示 + 商业承接”表达：

- 先解释品牌页在平台中的角色
- 加入品牌分类层
- 品牌卡片改成企业名片式结构
- 明确分类、主营方向、地区、城市和“已收录 / 可参考”感

### 6.5 头部语义修正

站点头部也同步修正：

- 不再使用“预留频道”语义
- 改为“信任与参考结构”
- 顶部说明同步强化知识入口、信任入口和服务入口

## 7. 本轮优点

- 首页终于不再被内容流主导
- 用户更容易在第一眼理解中眠网是什么
- 七个总入口已经显性成立
- 词库页更像词条系统，不再像内容频道
- 知识页更像问题入口，不再像资讯页
- 品牌页更像行业名录和合作入口，不再像普通展示卡片集合

## 8. 本轮缺点与代价

- 榜单、指数、标准、智库当前仍然主要通过首页结构表达，尚未落成独立模块
- 这一轮优先调整结构，细节视觉和交互还没有进一步打磨
- 内容流被主动降权后，首页的“媒体感”会下降，这是有意取舍
- 由于当前仍是 demo 数据，行业入口结构已经成立，但真实数据承接尚未跟上

## 10. 本轮结论

本轮最重要的结论是：

中眠网首页必须先表达行业入口结构，再表达内容流；只有这样，平台才不会继续被误读成普通资讯站。

本轮确认的关键判断：

- 首页的主语应该是“入口结构”，不是“推荐内容”
- 词库页的主语应该是“概念节点”，不是“卡片内容”
- 知识页的主语应该是“问题解决”，不是“文章聚合”
- 品牌页的主语应该是“行业名录与信任展示”，不是“品牌推荐”

## 11. 下一轮建议

下一步最值得做的事情建议如下：

1. 继续补齐榜单、指数、标准、智库的前台结构页，让首页中的信任体系真正落地
2. 再把前台详情页与真实数据接上，让知识底座从结构成立走向数据成立
3. 最后再进入后台真实表单与发布流转，把前后台真正打通
