# 版本记录：v0.1-中眠网一期技术实施方案基线

## 1. 本轮目标

本轮目标是为“中眠网”建立第一阶段的产品与技术基线，不直接进入业务编码，而是先明确：

- 一期产品边界与 MVP 上线范围
- 前台门户与后台 CMS 的模块划分
- 核心数据模型与内容关系
- Next.js + PostgreSQL + Prisma + Tailwind CSS 的落地方式
- 便于后续扩展榜单、指数、联盟、大会、智能体等模块的目录骨架

本轮属于：

- 功能补充前的架构设计
- 可维护性提升
- 实验探索前的正式基线建立

中眠网不是普通资讯站，而是：

“中国睡眠产业的基础知识入口 + 信任入口 + 商业入口”

这意味着一期设计不能只围绕“发文章”，而应同时支撑：

- 知识组织
- 词条沉淀
- 品牌与企业展示
- 内容运营
- SEO/GEO 友好扩展

## 3. 问题分析

### 3.1 表面目标

用户希望参考传统垂直行业门户的产品逻辑，先完成中眠网的一期产品与技术基础搭建方案，并输出可开发版本。

### 3.2 核心矛盾

真正的核心矛盾不是“如何先搭个站”，而是：

- 如何在一期就把内容门户、知识基础设施和未来商业入口统一到一个可扩展底座上
- 如何既保证首页信息密度高，又不让信息架构混乱
- 如何让后台不只是 CMS，而是支持未来 AI 编辑部工作流
- 如何让数据模型从一开始就兼容内容、词条、品牌、产品、专题、榜单、指数等不同实体

### 3.3 设计重点

本轮优先解决以下问题：

1. 统一内容与实体模型，避免后续重复建模
2. 让前台栏目与后台管理结构一一对应，降低运营成本
3. 让 SEO / GEO 友好不是补丁，而是路由与字段层面的默认能力
4. 预留未来商业化入口，但一期只做必要骨架，不提前过度实现

## 4. 候选方案比较

### 方案 A：传统资讯站优先，后续再补知识库与品牌库

**核心思路**

先按新闻门户方式建设首页、栏目页、文章页，词库和品牌库以后再单独追加。

**优点**

- 上线速度快
- 首页内容感容易做出来
- 编辑部流程简单

**缺点**

- 数据模型后续大概率要推翻
- 词条、知识、品牌之间关系难以统一
- 未来榜单、指数、专题只能继续叠补丁

**风险**

- 一期看似轻，但二期重构成本高
- SEO/GEO 结构会被文章中心化思路限制

**结论**

不选。它适合资讯站，不适合“基础知识入口 + 信任入口 + 商业入口”。

### 方案 B：知识图谱优先，先做非常重的数据中台

**核心思路**

在一期就建立较复杂的实体关系、知识图谱、审核流、内容流水线和指标体系。

**优点**

- 理论扩展性强
- 数据结构统一度高
- 未来高级功能空间大

**缺点**

- 首期开发周期过长
- 后台复杂度高，难以快速运营
- 过早设计会引入大量未验证抽象

**风险**

- 空仓库阶段直接上重平台，容易出现“架构先行、内容滞后”
- 团队后续若资源有限，可能难以完成真正落地

**结论**

不选。扩展性很好，但不适合一期快速上线。

### 方案 C：门户前台 + 统一内容实体底座 + 轻量 AI 编辑部

**核心思路**

一期围绕门户首页、词库、知识库、品牌库与基础后台来设计；数据模型从一开始统一实体与内容关系，但后台流程保持轻量，AI 编辑部只做基础链路。

**优点**

- 能兼顾上线效率与后续扩展
- 前台能形成行业门户感
- 后台能承接未来商业模块
- SEO/GEO 与内容结构能同步设计

**缺点**

- 需要在“不过度设计”和“足够抽象”之间保持克制
- 首页和后台信息架构需要一次设计清楚

**风险**

- 若栏目边界定义不清，后续仍可能出现实体重复录入

**结论**

最终采用本方案。

## 5. 最终决策

### 5.1 一期总体产品策略

中眠网一期采用“门户首页驱动 + 内容知识双轮 + 品牌展示 + 后台运营底座”的方式推进。

其中：

- 首页负责行业入口、内容聚合、品牌曝光和未来商业入口预热
- 睡眠词库承担“概念解释与 SEO 长尾入口”
- 睡眠知识承担“问题解答、专题知识与教育内容”
- 睡眠品牌承担“信任入口与商业展示基础”
- 后台 CMS 统一管理内容、词条、品牌、分类、标签、SEO、发布状态
- AI 编辑部承担“选题到发布”的基础工作流

### 5.2 技术架构决策

采用以下技术组合：

- 前端框架：Next.js App Router
- 样式系统：Tailwind CSS
- 数据库：PostgreSQL
- ORM：Prisma
- 鉴权：NextAuth 或基于自定义 Session 的后台鉴权层
- 富文本：建议采用可扩展编辑器，如 Tiptap
- 文件存储：一期可先本地 / 对象存储抽象层，后续接 OSS / S3
- AI 能力：通过独立服务层封装模型调用，避免与 CMS 页面逻辑耦合

### 5.3 一期模块边界

一期实际实现：

- 首页门户
- 睡眠词库
- 睡眠知识库
- 睡眠品牌库
- 基础后台 CMS
- AI 编辑部基础版
- SEO / GEO 基础能力

一期只预留结构：

- 睡眠产品
- 中眠榜
- 中眠指数
- 睡眠标准
- 中眠智库

### 6.3 数据模型设计

#### 核心建模原则

一期避免为每类内容都单独造一套发布系统，而是采用“统一内容主表 + 垂直实体扩展表”设计。

#### 主要实体

1. `User`
- 后台用户
- 支持角色、状态、最近登录时间

2. `Role`
- 超级管理员
- 编辑
- 审核员
- 运营

3. `Permission`
- 细粒度后台权限点

4. `Category`
- 栏目分类
- 支持层级结构
- 用于知识、资讯、品牌、产品等不同资源归类

5. `Tag`
- 通用标签

6. `Content`
- 统一内容主表
- 支持内容类型：`article`、`knowledge`、`topic`、`report`、`guide`
- 维护标题、摘要、正文、封面、作者、状态、发布时间、SEO 字段

7. `ContentRevision`
- 内容版本记录
- 用于人工编辑与 AI 初稿的历史追踪

8. `Term`
- 睡眠词条
- 包含词条名、别名、定义、扩展解释、关联问题、关联内容、SEO 信息

9. `Brand`
- 睡眠品牌或企业
- 包含品牌名、简介、主营方向、地区、官网、联系方式、展示状态、推荐状态

10. `Product`
- 睡眠产品
- 一期只预留基础结构

11. `Topic`
- 专题聚合页
- 一期可先借助 `Content` 类型实现，后续再增强

12. `EditorialTopic`
- AI 编辑部选题池
- 包含题目、优先级、来源、状态、关键词

13. `EditorialSource`
- 资料录入
- 记录素材来源、原始摘要、附件链接、可信度备注

14. `EditorialDraft`
- AI 生成初稿
- 记录生成参数、模型信息、生成结果、人工处理状态

15. `WorkflowLog`
- 发布状态流转记录
- 记录谁在什么时间将状态从什么改到什么

16. `SeoMeta`
- 可复用 SEO / GEO 元信息
- 包括 title、description、keywords、canonical、structured data 预留字段

17. `FeaturedSlot`
- 首页推荐位 / 模块位
- 控制首页 Banner、推荐内容、推荐品牌、热点问题等内容编排

#### 关键关系

- `Content` 与 `Category` 多对多
- `Content` 与 `Tag` 多对多
- `Content` 与 `Term` 多对多
- `Content` 与 `Brand` 可多对多
- `Brand` 与 `Product` 一对多
- `Content` 与 `ContentRevision` 一对多
- `EditorialTopic` 与 `EditorialSource` 一对多
- `EditorialTopic` 与 `EditorialDraft` 一对多
- `WorkflowLog` 可关联 `Content`、`Term`、`Brand`

### 6.9 页面骨架建议

#### 首页信息架构

首页建议采用“高信息密度但分区清晰”的门户布局：

1. 顶部工具栏
2. 主导航
3. 门户主 Banner 与今日推荐
4. 核心入口矩阵
5. 热门问题区
6. 推荐知识 / 资讯双列区
7. 推荐品牌区
8. 最新内容流
9. 专题 / 榜单 / 指数预告入口
10. 页脚

#### 栏目视觉方向

不照搬传统旧门户视觉，但保留专业、可信、信息型产品特征：

- 版式上强调栅格与信息模块
- 颜色上偏理性、健康、可信赖
- 字体与留白控制“专业感”，避免资讯站噪音感
- 首页 Banner 不追求大促销风，而应强化行业基础设施定位

## 10. 本轮结论

本轮最重要的结论是：

中眠网一期应采用“门户前台 + 统一内容实体底座 + 轻量 AI 编辑部 + 可扩展后台”的模式，而不是单纯资讯站模式。

已确认的判断：

- 一期必须优先打好内容、词条、品牌三类核心资源的统一底座
- SEO / GEO 必须成为默认设计能力，而非后期补丁
- 后台应以运营闭环为中心，而不是只做静态录入系统

尚待后续确认的部分：

- 后台鉴权采用 NextAuth 还是自定义 Session
- 富文本编辑器的具体选择
- 首页详细视觉方向
- 是否在一期加入搜索页与站内检索增强

2. 紧接着落地 Prisma Schema 与后台权限模型，确保内容、词条、品牌三类资源先跑通
