# 中眠网上线执行清单

## 1. 生产环境变量

上线前必须配置以下环境变量：

- `NODE_ENV=production`
- `DATABASE_URL`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_SESSION_SECRET`
- `NEXT_PUBLIC_SITE_URL`

推荐示例：

```env
NODE_ENV=production
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/zhongmian?schema=public"
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="请改成正式强密码"
ADMIN_SESSION_SECRET="请改成足够长的随机密钥"
NEXT_PUBLIC_SITE_URL="https://你的正式域名"
```

执行要求：

1. 不使用开发环境默认账号密码
2. 不把 `.env.local` 或生产密钥提交到仓库
3. `NEXT_PUBLIC_SITE_URL` 必须与正式域名一致
4. `ADMIN_SESSION_SECRET` 必须单独生成，不能复用开发值

## 2. 生产数据库

上线前准备：

1. 创建生产 PostgreSQL 数据库
2. 创建专用数据库账号
3. 给该账号授予目标数据库访问权限
4. 确认部署环境可访问数据库

推荐步骤：

1. 创建数据库：`zhongmian`
2. 创建数据库账号
3. 将生产数据库连接串写入 `DATABASE_URL`
4. 首次初始化数据库结构
5. 初始化基础 seed 数据

执行命令：

```bash
npx prisma db push
npm run db:seed
```

上线前检查：

- 数据库可连接
- Prisma 可正常读写
- Category / Tag 等基础数据已初始化
- 首批公开内容已存在
- 明显测试残留没有进入 `PUBLISHED`

## 3. 域名与 HTTPS

必须准备：

1. 正式域名
2. DNS 解析
3. HTTPS 证书
4. 域名与站点配置一致

建议口径：

- 前台：`https://你的域名`
- 后台：`https://你的域名/admin`

检查项：

- 域名已解析到部署服务
- HTTPS 可正常访问
- 非 HTTPS 访问自动跳转到 HTTPS
- sitemap、metadata 中没有 `localhost`
- 后台登录页可通过正式域名访问

## 4. 部署步骤

按以下顺序执行：

1. 拉取远程最新代码
2. 安装依赖
3. 配置生产环境变量
4. 连接生产数据库
5. 执行 Prisma 初始化
6. 执行构建
7. 启动正式服务
8. 绑定域名与 HTTPS
9. 执行上线前冒烟测试

命令参考：

```bash
git pull --ff-only
npm install
npx prisma db push
npm run db:seed
npm run build
npm run start
```

如使用部署平台：

- Build Command：`npm run build`
- Start Command：`npm run start`
