# 中眠网上线执行清单

## 一、环境变量

生产环境至少准备：

```env
NODE_ENV=production
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/zhongmian?schema=public"
ADMIN_USERNAME="正式后台账号"
ADMIN_PASSWORD="正式后台密码"
ADMIN_SESSION_SECRET="长度足够且随机的密钥"
NEXT_PUBLIC_SITE_URL="https://你的正式域名"
```

检查项：
- [ ] `DATABASE_URL` 指向生产 PostgreSQL
- [ ] `ADMIN_SESSION_SECRET` 不使用开发默认值
- [ ] 后台正式账号密码已确认
- [ ] 生产环境变量配置在部署平台，不放入仓库

## 二、生产数据库

执行顺序：
1. 创建生产数据库 `zhongmian`
2. 创建专用数据库账号并授权
3. 配置生产 `DATABASE_URL`
4. 执行 schema 同步
5. 导入首批正式内容或运行最小 seed

推荐命令：

```bash
npx prisma db push
npm run db:seed
```

检查项：
- [ ] 数据库可从部署环境访问
- [ ] 扩展栏目 schema 已同步
- [ ] 首批正式内容已准备
- [ ] 没有测试内容误发布

## 三、域名与 HTTPS

检查项：
- [ ] 正式域名可访问
- [ ] 已配置 HTTPS
- [ ] 非 HTTPS 会自动跳转 HTTPS
- [ ] `NEXT_PUBLIC_SITE_URL` 与正式域名一致
- [ ] sitemap、metadata 中没有 `localhost`

推荐口径：
- 前台：`https://你的域名`
- 后台：`https://你的域名/admin/login`

## 四、部署步骤

按顺序执行：

1. 拉取远程最新 `main`
2. 安装依赖
3. 配置生产环境变量
4. 同步数据库 schema
5. 准备首批内容
6. 构建项目
7. 启动正式服务
8. 绑定域名与 HTTPS
9. 做上线前冒烟测试
10. 再正式开放访问

参考命令：

```bash
npm install
npx prisma db push
npm run db:seed
npm run build
npm run start
```

## 六、上线后第一轮检查

建议上线后 30 分钟内人工复查：

1. 首页与各栏目页是否正常打开
2. 随机抽查三条已发布详情页
3. 后台是否仍能登录
4. 保存一条草稿并确认不前台暴露
5. 发布一条内容并确认前台出现
6. 下线一条内容并确认前台消失
7. AI 编辑部是否仍能创建任务并挂接草稿
8. 查看日志中是否有持续 Prisma / 鉴权 / 500 错误
